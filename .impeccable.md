# Design Context — Nurea Parfums

## Target Audience
Clientèle francophone, 18-45 ans, passionnée de parfumerie de niche et de luxe. Mobile-first (80%+ du trafic). Connaisseurs et curieux cherchant des fragrances rares, pas du mass-market.

## Use Cases
- Parcourir le catalogue (filtres, recherche floue, exploration par maison/univers)
- Découvrir des maisons de niche peu connues
- Contacter la conciergerie via WhatsApp/Snapchat pour commander
- Admin : gérer le catalogue (CRUD parfums/marques) depuis un mobile

## Brand Personality & Tone
**Nuit Bordeaux** — luxe éditorial sombre, intimiste, confidentiel. Pas tape-à-l'oeil, pas criard. La sophistication vient du vide, du rythme typographique, de la retenue. Cuivre et bordeaux sur fond quasi-noir. Serif GFS Didot pour les titres (haute couture), Inter pour le corps (lisibilité).

## Aesthetic Direction
- **Tone** : Luxury editorial / dark cinematic — proche des codes de la haute parfumerie
- **Memorable hook** : Le grain texturé omniprésent + les transitions ease-out-expo qui donnent une impression de fluidité organique
- **Constraint** : Next.js 16 App Router, Tailwind CSS, variables CSS --nurea-*, pas de librairie UI externe

# Permanent Design & Engineering Rules

## Typography
- Display : GFS Didot (serif haute couture) — titres, prix, noms de parfums
- Body : Inter (lisibilité) — corps, labels, navigation
- Scale : clamp() fluid sizing, tracking resserré sur les titres, élargi sur les labels uppercase
- Line-height : 1.1-1.25 titres, 1.5-1.65 corps
- Never : system-ui seul, Poppins, Roboto en display

## Color System (tokenisé dans globals.css)
- Dark (default) : fond #0A0508, surface #140E12, accent #C46A6A, cuivre #C4956A, texte #F0E6E0
- Light : fond #FAF6F2, surface #FFFFFF, accent #8B3A3A
- Ratio 60/30/10 : fond dominant / surface secondaire / accent bordeaux-cuivre
- Pas de noir pur (#000), pas de blanc pur (#fff) — toujours teinté
- Pas de cyan-on-dark, pas de gradient violet-bleu, pas de néon

## Layout & Space
- Mobile-first, scale up avec intention
- Rythme spatial varié : groupements serrés, séparations généreuses
- Briser la grille avec intention (asymétrie, éléments décalés)
- Pas de cartes dans des cartes — aplatir la hiérarchie
- Safe-area padding sur iOS (env(safe-area-inset-*))

## Motion
- Easing : ease-out-expo (cubic-bezier(0.16, 1, 0.3, 1)) pour toutes les transitions
- Durées : 150ms micro-interactions, 300ms transitions, 600-800ms reveals
- Scroll reveals avec IntersectionObserver, staggered children
- prefers-reduced-motion respecté
- Jamais : bounce, elastic, animations sur width/height

## Dark UI
- 4 niveaux de surface : bg (#0A0508) / surface (#140E12) / surface-hover (#1C1418) / overlay
- 4 niveaux de texte : text (#F0E6E0) / muted (#B89AA8) / subtle (#6B5060) / disabled
- Bordures à ~12% d'opacité bordeaux, hover à ~30%
- Glow subtil sur les accents (rgba accent à 6% d'opacité)
- Texte ivoire, jamais blanc pur

## Accessibility
- Contraste minimum 4.5:1 sur tout le texte
- Zones tactiles minimum 44x44px (min-h-11 min-w-11)
- Focus rings visibles (ring-2 ring-[var(--nurea-accent)])
- aria-label sur tous les boutons icône-seuls
- Tab order = ordre visuel
- Labels sur tous les champs de formulaire

## Components
- Composables, accessibles, variant-based
- Logique (hooks) séparée de la présentation
- Lucide React pour les icônes (SVG, jamais emoji)
- clsx pour la composition conditionnelle de classes

## Performance
- Images : next/image avec dimensions explicites, lazy loading
- Fonts : next/font/google avec font-display: swap
- Pas d'import inutilisé
- Squelettes d'attente (pulse) pour le contenu asynchrone

## Copy & UX Writing
- Français, direct, jamais mièvre
- Boutons = verbes d'action : "Enregistrer", "Créer", "Explorer"
- Messages d'erreur : ce qui s'est passé + quoi faire ensuite
- Empty states qui guident, pas juste "rien ici"
- Phrases courtes. Voix active. Pas de remplissage.
- Interdit : "N'hésitez pas", "Bienvenue sur", "Cliquez ici", formules creuses

## Self-Audit (avant chaque output)
1. Un designer senior chez Linear/Vercel approuverait-il ça ?
2. Est-ce que ça ressemble à du générique 2024 ? → refaire
3. Y a-t-il un élément mémorable ?
4. Tous les états sont-ils couverts (vide, chargement, erreur, succès) ?
5. Le système de couleurs est-il sémantique et tokenisé ?
6. Tout est-il accessible au clavier ?
